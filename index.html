Jeu du Moulin (PyGame)

Le projet est un jeu du Moulin développé en Python avec Pygame, où le joueur affronte une IA. Le jeu se déroule en trois phases :

  1. Phase de Placement : Les joueurs placent alternativement leurs pions sur le plateau. L'objectif est de former des lignes de trois pions (un moulin) pour retirer un pion adverse.

  2. Phase de Mouvement : Une fois tous les pions placés, les joueurs déplacent leurs pions pour former des moulins. Les joueurs peuvent retirer des pions adverses lorsqu'un moulin est créé. Si un joueur n’a plus que trois pions, il n’est plus limité dans ses mouvements et peut déplacer ses pions librement sur le plateau.

  3. Phase de Fin de Partie : Lorsqu'un joueur n'a plus que deux pions ou ne peut plus déplacer ses pions, la partie se termine. Le gagnant est celui qui reste avec des pions en jeu.

Le jeu utilise une interface graphique simple, offrant une expérience interactive avec l'IA comme adversaire.

Base de données

Base de données sur les JO2024

Le projet consistait à développer une base de données sur les Jeux Olympiques 2024, alimentée par des scripts Python de web scraping pour collecter les données automatiquement. La base de données était gérée avec PostgreSQL et mise à disposition via une interface web pour permettre la consultation des informations. Un serveur Falcon en Python a été utilisé pour gérer les interactions entre l'interface utilisateur et la base de données, tandis qu'un serveur Apache hébergeait le site web. Ce système a permis de centraliser les données des JO2024 et de les rendre facilement accessibles. Ce projet a été réalisé en groupe.

Projet actuel | Doors Keeper

Styles de jeu : 3D, Co-op, Roguelike, Adventure

Le projet est un jeu 3D coopératif de type roguelike, où les joueurs incarnent des employés d'une entreprise mystérieuse chargés de gérer des portes menant à des donjons générés aléatoirement. À l'intérieur, ils doivent accomplir des missions tout en récupérant des objets à vendre pour gagner des crédits. Ces crédits permettent d'acheter de l'équipement sur un ordinateur et de choisir de nouvelles portes. Les joueurs doivent éviter de perdre trop de crédits, car échouer à une mission entraîne des pertes, et le but est de maintenir un solde positif.

Ce projet utilise Mirror, une API de réseau pour Unity, et FuzzySteam, une solution d'intégration Steam, pour implémenter le multijoueur
